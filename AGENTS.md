# Agents Overview

A catalog of available agents in `subagents/` with purpose, highlights, and paths.

## Directory

- ai-sdk-prompt-architect — Architect AI SDK integrations, prompts, and agent workflows
- app-replication-architect — Analyze apps and design replication strategies
- code-reviewer-typescript-nextjs — Expert TS/Next.js/MUI code reviews
- dialogue-story-specialist — Narrative/dialogue creation with structure
- flutter-ux-architect — Flutter UX plus NextJS auth integration guidance
- fsd-architecture-specialist — Feature-Sliced Design guidance and migrations
- game-design-architect — Game design specs and systems
- mui-design-expert — MUI v5+ and Next.js app router UI expert
- nextjs-fsd-developer — NextJS developer using Feature-Sliced Design
- product-marketing-strategist — Insights, prioritization, and messaging
- project-manager — Phased plans, tasks, stage gates, and approvals
- psychology-consultant — Evidence-based mental health UX consultation
- technical-integration-architect — Integration strategies across stacks
- telegram-bot-architect — Telegram bots/mini apps architecture and code

## How to use

1. Open the agent file under `subagents/`.
2. Read the scope and deliverables; follow any working directory conventions (often `.agents-playbook/[task-name]/`).
3. Produce outputs as specified and cross-link from related docs.
